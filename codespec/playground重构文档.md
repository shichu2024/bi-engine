## 一、playground项目概述
### 1.1 playground项目定位
本项目是BI组件库的**官方交互式文档、开发调试工具、视觉回归测试一体化平台**，为前端开发者、BI报表工程师提供「组件浏览-场景体验-配置编辑-实时预览-样式合规测试」的全闭环能力，同时解决BI组件库迭代过程中图表样式回归（文本截断、布局错位、颜色偏移、元素缺失等）的核心痛点。

### 1.2 核心目标
1. **降低使用门槛**：对标Ant Design组件文档的交互体验，提供开箱即用的组件浏览与场景示例，让使用者快速掌握BI组件的配置与用法。
2. **提升开发效率**：通过DSL实时编辑+双向同步渲染，实现BI组件配置的所见即所得，支持配置一键复制/导出，无缝对接业务项目。
3. **管控样式稳定性**：搭建自动化视觉回归测试（VRT）体系，通过标准化截图对比，提前发现组件迭代中的非预期样式变更，保障BI图表在各类边界场景下的渲染一致性。
4. **标准化测试流程**：将组件测试用例与业务场景深度绑定，实现测试用例的可复用、可追溯、可批量执行，融入组件库CI/CD流水线。

### 1.3 目标用户
| 用户角色 | 核心使用场景 |
|----------|--------------|
| 前端开发工程师 | 组件API学习、配置调试、组件库迭代开发与自测 |
| BI报表开发工程师 | 图表场景复用、DSL配置调试、报表样式效果验证 |
| 测试工程师 | 组件视觉回归测试、批量用例执行、回归问题定位 |
| UI设计师 | 组件样式还原度验证、主题效果预览、设计规范落地校验 |
| 产品经理 | BI组件能力调研、业务场景方案验证 |

### 1.4 术语定义
| 术语 | 全称 | 定义 |
|------|------|------|
| Playground | 交互训练场 | 本项目的核心载体，提供BI组件的浏览、编辑、预览、测试一体化环境 |
| DSL | 领域特定语言 | 本项目中指BI组件的标准化配置描述语言，基于JSON格式定义，用于描述组件的类型、数据、样式、交互行为 |
| VRT | 视觉回归测试 | Visual Regression Testing，通过像素级对比组件渲染截图与基准图的差异，识别非预期样式变更的测试方式 |
| 基准图（Baseline） | 基准截图 | 组件在指定场景、环境下的标准渲染截图，作为视觉对比的唯一合规标准 |
| 测试用例（Case） | 测试用例 | 视觉回归测试的最小执行单元，包含关联组件、DSL配置、Mock数据、测试环境参数、忽略区域配置等完整信息 |

整体采用**经典三栏式布局**，对标Ant Design组件文档的交互逻辑，同时支持布局模式动态切换，全局布局结构如下：

| 区域 | 固定位置 | 宽度规则 | 核心功能 |

| 主内容区 | 页面右侧，自适应宽度 | 占据剩余视口宽度，最小宽度800px | 承载「场景演示」「DSL编辑+预览」「测试中心」三大核心视图 |

### 2.3 核心用户流程
#### 流程1：组件浏览与场景体验流程
1. 用户进入Playground，左侧列表默认展开「基础图表」分类，主内容区展示首个组件的场景演示页
2. 用户可通过分类折叠/展开、搜索框检索，快速定位目标组件
3. 点击目标组件，主内容区切换至该组件的概览与多场景演示
4. 用户点击场景卡片，展开可交互的组件预览、DSL代码片段，支持一键复制配置

#### 流程2：DSL编辑与实时预览流程
1. 用户在场景演示页点击「在编辑器中打开」，主内容区切换为「DSL编辑器+渲染预览」左右分栏模式
2. 左侧编辑器自动加载当前场景的DSL配置，右侧实时渲染对应组件效果
3. 用户修改DSL，右侧预览区防抖同步更新渲染结果，支持语法校验、自动补全、格式化
4. 用户完成配置后，可一键复制DSL、导出配置文件、保存为测试用例

#### 流程3：自动化视觉回归测试流程
1. 用户在编辑器/场景演示页，将当前配置保存为测试用例，填写用例信息、配置测试环境参数
2. 点击「生成基准图」，系统锁定环境与数据，生成标准化基准截图并与用例绑定
3. 组件库迭代后，用户可单条/批量执行测试用例，系统自动生成新截图并与基准图做像素级对比
4. 测试完成后，生成可视化测试报告，标记差异用例，支持查看差异详情、一键更新基准图、跳转编辑器修复问题

#### 流程4：CI/CD流水线集成测试流程
1. 开发人员提交组件库代码PR，自动触发CI流水线
2. 流水线调用Playground CLI工具，执行全量视觉回归测试用例
3. 测试完成后，自动生成测试报告，若存在失败用例，直接在PR中评论报告链接与差异详情
4. 测试通过方可合并代码，不通过则阻断合并，通知开发人员修复

## 三、核心功能模块详细需求
### 模块1：全局导航与公共能力
#### 1.1 顶部导航栏
| 功能点 | 界面要求 | 交互规则 |
|--------|----------|----------|
| 品牌标识 | 左侧展示组件库Logo与名称「XX BI 组件库」 | 点击跳转至Playground首页 |
| 版本切换器 | Logo右侧展示下拉选择器，默认选中当前最新版本 | 1. 下拉列表展示组件库所有正式版本，支持切换<br>2. 切换版本后，自动加载对应版本的组件列表、场景示例、基准图<br>3. 开发版本标记「beta」标识，废弃版本标记「deprecated」标识 |
| 主题切换 | 导航栏中部，提供亮色/暗色/自定义主题切换开关 | 1. 切换主题后，全局界面与组件渲染区同步生效<br>2. 主题选择本地缓存，用户下次打开自动沿用上次选择 |
| 核心入口 | 导航栏右侧，依次排列「API文档」「测试中心」「GitHub」按钮 | 1. 「API文档」跳转至组件库官方参数文档站点<br>2. 「测试中心」跳转至独立的测试管理页面<br>3. 「GitHub」跳转至组件库代码仓库 |
| 全局搜索 | 导航栏右侧，提供全站点搜索框 | 支持搜索组件名称、场景名称、API参数、用例名称，搜索结果支持快速跳转 |

#### 1.2 全局公共能力
1. **响应式适配**
   - 桌面端（≥1200px）：完整展示三栏布局，左侧列表可折叠
   - 平板端（768px-1200px）：左侧列表默认折叠，点击展开为抽屉式浮层，主内容区自适应
   - 移动端（<768px）：隐藏左侧列表，仅保留顶部导航与主内容区，通过汉堡菜单唤起组件列表抽屉
2. **全局快捷键**
   - `Ctrl/Cmd + K`：聚焦全局搜索框
   - `Ctrl/Cmd + S`：保存当前DSL配置/测试用例
   - `Ctrl/Cmd + B`：折叠/展开左侧组件列表
   - `Ctrl/Cmd + Enter`：手动触发DSL渲染更新
   - `Ctrl/Cmd + R`：运行当前用例的视觉回归测试
3. **全局缓存机制**
   - 本地缓存用户最近访问的组件、编辑的DSL配置、主题选择、版本选择
   - 刷新页面后，自动恢复至用户上次离开时的页面状态

### 模块2：左侧组件导航与检索系统
#### 2.1 组件分类管理
- 按BI组件的业务属性与使用频率，做二级分类管理，一级分类默认包含：
  1. 基础图表：折线图、柱状图、饼图、环形图、散点图、面积图
  2. 高级分析图表：热力图、桑基图、漏斗图、雷达图、水波图、仪表盘、 treemap
  3. 表格组件：基础表格、交叉表、透视表、树形表格
  4. 布局容器：卡片、网格布局、标签页、折叠面板、分栏容器
  5. 筛选交互组件：日期范围选择器、下拉筛选器、数值输入框、单选/复选框、搜索框
  6. 文本展示组件：标题、指标卡、富文本、备注框
- 支持一级分类的展开/折叠，默认展开「基础图表」分类，折叠状态本地缓存
- 分类支持权限管控，内部定制组件分类仅对内部用户可见

#### 2.2 组件检索与列表展示
1. **组件搜索**
   - 分类顶部固定搜索框，支持输入组件名称、关键词、描述实时过滤组件列表
   - 搜索结果高亮匹配关键词，支持拼音模糊匹配，无匹配结果时展示「暂无匹配组件」提示
   - 搜索响应时间≤200ms，支持回车快速选中首个匹配结果
2. **组件列表项展示**
   - 每个组件项包含：组件图标、组件名称、一句话功能描述
   - 组件项右侧展示测试状态标记：✅ 全部用例通过、❌ 存在失败用例、⚪ 无测试用例
   - 鼠标hover组件项时，展示浮层提示，包含组件核心能力与适用场景
   - 点击组件项，主内容区切换至该组件的场景演示页，组件项高亮选中状态，与主内容区双向联动

### 模块3：组件场景演示中心（主内容区默认视图）
主内容区默认展示当前选中组件的场景演示页，分为「组件概览区」和「场景卡片列表区」两部分。

#### 3.1 组件概览区
- 顶部展示组件全称、版本标记、详细功能说明、适用场景、不适用场景
- 提供「查看完整API文档」按钮，点击跳转至该组件的详细参数文档页
- 展示组件核心能力标签，如「支持钻取」「响应式适配」「多主题兼容」

#### 3.2 场景卡片列表区
- 每个组件预置≥3个典型业务场景，覆盖基础用法、进阶用法、边界场景，场景标题清晰明确，如「折线图-基础时间趋势」「折线图-多系列对比」「折线图-超长文本标签截断测试」
- 每个场景卡片包含：场景标题、简短描述、组件预览缩略图
- 点击场景卡片，下方展开对应场景的详情面板，包含3个核心区域：
  1. **交互预览区**：嵌入完整可交互的组件实例，支持组件原生的筛选、钻取、缩放、tooltip、图例切换等所有交互能力，与业务项目使用效果完全一致
  2. **DSL代码片段区**：展示该场景对应的完整DSL配置，支持语法高亮、行号展示，右上角提供「一键复制」按钮，复制成功后展示成功提示
  3. **操作工具栏**：提供「在编辑器中打开」「运行单场景测试」「生成基准图」三个核心按钮
     - 点击「在编辑器中打开」：主内容区切换至DSL编辑模式，自动加载当前场景的DSL配置
     - 点击「运行单场景测试」：执行当前场景对应的测试用例，实时返回测试结果
     - 点击「生成基准图」：将当前场景的渲染结果生成为基准图，绑定至对应测试用例

### 模块4：DSL编辑器与实时渲染预览系统（编辑模式）
点击「在编辑器中打开」后，主内容区切换为**左右等分可拖拽分栏布局**，左侧为DSL编辑器，右侧为实时渲染预览区，分栏支持拖拽调整宽度，最小宽度占比30%，最大占比70%。

#### 4.1 左侧DSL编辑器
1. **编辑器核心能力**
   - 基于Monaco Editor（VS Code内核）实现，与VS Code操作体验完全一致
   - 支持JSON格式DSL的语法高亮、代码折叠、行号展示、右键菜单
   - 智能自动补全：实时提示组件类型、参数名、枚举值、数据结构，补全响应时间≤300ms
   - 实时语法校验：对JSON格式错误、参数类型错误、必填项缺失进行实时标红，hover时展示详细错误提示
   - 支持代码格式化：一键美化DSL结构，统一缩进与换行规范
   - 支持大文本编辑优化，DSL配置行数≥1000行时无明显卡顿
2. **编辑器工具栏**
   - 固定在编辑器顶部，提供以下操作按钮，从左到右依次为：
     1. 「重置」：恢复至进入编辑器时的初始DSL配置，二次确认后执行
     2. 「格式化」：一键格式化DSL代码
     3. 「复制DSL」：一键复制完整DSL配置，复制成功提示
     4. 「导出配置」：将当前DSL导出为JSON文件，文件名为「组件名-场景名-时间戳.json」
     5. 「保存为测试用例」：弹出弹窗，填写用例信息后，将当前DSL与配置保存为新的测试用例
     6. 「运行测试」：执行当前DSL对应的视觉回归测试，实时返回测试结果
3. **辅助能力**
   - 支持DSL修改历史记录，本地缓存最近10次修改，支持一键回退至上一版本
   - 支持编辑器字号调整、换行模式切换、空格/制表符缩进切换，配置本地缓存

#### 4.2 右侧实时渲染预览区
1. **实时同步渲染**
   - 左侧DSL修改后，通过300ms防抖触发右侧重新渲染，确保修改与渲染同步无明显延迟
   - 渲染过程中展示加载动画，渲染失败时展示友好的错误提示，明确标注错误原因与位置（如「DSL参数错误：xAxis.data 应为数组类型，当前为字符串」）
   - 渲染环境与业务项目完全一致，确保预览效果与线上效果100%还原
2. **渲染交互能力**
   - 渲染的组件支持完整的原生交互能力，与场景演示区的交互体验一致
   - 预览区顶部工具栏提供：
     1. 「全屏预览」：点击后隐藏编辑器，全屏展示组件渲染效果，按ESC退出全屏
     2. 「主题切换」：与全局主题联动，支持快速切换亮色/暗色主题，预览组件在不同主题下的效果
     3. 「视口切换」：提供预设视口选项（PC 1920px、平板 1366px、移动端 375px），支持自定义视口尺寸，测试组件响应式适配效果
     4. 「禁用动画」：开关控件，开启后禁用组件所有过渡动画、tooltip效果，确保截图时无动态内容干扰
3. **差异预览能力**
   - 若当前DSL对应的测试用例执行后存在差异，预览区底部展示「差异预览」开关
   - 开启开关后，预览区直接高亮渲染差异区域，支持切换「基准图叠加」「差异红框标记」两种预览模式

### 模块5：自动化视觉回归测试系统（核心增强模块）
本模块为BI图表样式管控的核心模块，深度集成至Playground全流程，同时提供独立的测试中心页面做全量管理。

#### 5.1 测试用例管理
测试用例是视觉回归测试的最小执行单元，与组件场景深度绑定，分为「系统预置用例」和「用户自定义用例」两类。
1. **用例创建规则**
   - 系统预置用例：每个组件的默认场景自动创建1条基础测试用例，不可删除，仅可更新基准图
   - 用户自定义用例：用户在编辑器/场景演示页，可将任意DSL配置保存为自定义用例，支持编辑、删除
   - 用例创建必填项：用例名称、关联组件、DSL配置、固定Mock数据
   - 用例创建选填项：用例标签、优先级、测试环境配置、忽略区域配置、备注说明
2. **用例属性配置**
   - **固定Mock数据**：强制绑定静态Mock数据，禁止使用动态随机数据、实时时间戳，确保每次渲染的数据完全一致，避免数据差异导致的截图误差
   - **测试环境配置**：支持配置多套测试环境参数，包括：
     - 视口尺寸：支持配置多个尺寸，如1920×1080、1366×768、375×667
     - 主题类型：亮色/暗色/自定义主题
     - 浏览器类型：Chrome/Firefox/Safari
     - 设备像素比：1x/2x/3x
   - **忽略区域配置**：支持在截图上可视化框选动态内容区域（如随机水印、实时时间戳），对比时自动忽略该区域的像素差异，支持矩形框选、多边形框选
   - **标签与优先级**：支持给用例打标签（如「基础测试」「边界测试」「文本截断测试」「主题兼容测试」），设置P0-P3优先级，用于筛选与批量执行
3. **用例管理能力**
   - 支持按组件、标签、优先级、测试状态、创建人筛选用例
   - 支持用例批量编辑、批量删除、批量移动分类
   - 支持用例导入/导出，实现团队间用例复用
   - 用例修改留痕，记录创建人、修改人、修改时间、修改内容，支持追溯

#### 5.2 基准图（Baseline）管理
基准图是视觉对比的唯一标准，需确保生成环境的一致性与可追溯性。
1. **基准图生成规则**
   - 生成基准图时，系统自动执行以下操作：
     1. 锁定用例配置的视口尺寸、主题、浏览器、DPI等环境参数
     2. 强制禁用组件所有过渡动画、tooltip悬浮效果、动态加载效果
     3. 等待组件完全渲染完成后，对渲染区进行全尺寸高清截图，支持滚动截取长表格、长图表
     4. 生成的截图与组件库版本、用例ID、环境参数唯一绑定
   - 支持单条用例生成基准图，也支持在测试中心批量选择用例一键生成全量基准图
2. **基准图版本管理**
   - 基准图与组件库版本强绑定，每个大版本对应一套独立的基准图体系
   - 组件库小版本迭代时，支持基于上一版本基准图做增量更新
   - 保留基准图的历史版本，支持查看任意历史版本的基准图，支持一键回退至指定历史版本
   - 基准图更新需记录操作人、更新时间、更新原因，支持追溯
3. **基准图存储与同步**
   - 本地开发环境：基准图存储在浏览器IndexedDB中，支持本地导入/导出
   - 团队协作环境：基准图支持上传至云存储（AWS S3/阿里云OSS/企业私有存储），团队成员可一键拉取最新基准图，避免本地环境差异导致的对比误差
   - CI/CD环境：基准图存储在流水线制品库中，每次执行测试时自动拉取对应版本的基准图

#### 5.3 视觉对比引擎
核心为像素级对比+智能差异识别，兼顾准确性、性能与BI场景适配性。
1. **对比算法与能力**
   - 底层基于pixelmatch实现轻量级高性能像素级对比，基于Resemble.js实现差异可视化渲染
   - 支持配置**全局像素差异阈值**，默认允许0.1%以内的抗锯齿、字体渲染细微差异，避免误报
   - 支持单条用例自定义差异阈值，适配不同敏感度的测试场景
   - 支持配置忽略颜色差异、仅对比布局结构，适配主题切换场景的测试
2. **智能差异类型识别**
   - 自动识别并标记BI图表高频出现的差异类型，包括：
     - 文本截断异常：标签、标题、指标文本未按预期省略、换行、溢出
     - 布局偏移：图表坐标轴、柱子、图例、标签位置发生非预期移动
     - 颜色差异：主题色、文本色、背景色、渐变色发生非预期变更
     - 元素缺失：图例、坐标轴、tooltip、标注、数据点等元素丢失
     - 尺寸异常：组件宽高、内边距、元素大小发生非预期变更
   - 对不同差异类型进行分级，严重差异（元素缺失、文本溢出）优先展示
3. **对比性能要求**
   - 单张1920×1080分辨率截图的对比耗时≤500ms
   - 支持并发对比，默认并发数10，100条用例的全量对比耗时≤5min
   - 支持增量对比，仅执行DSL发生变更的用例，提升测试效率

#### 5.4 测试报告与差异展示
1. **测试报告概览**
   - 每次测试执行完成后，生成唯一的测试报告，包含：
     - 测试基础信息：执行时间、执行环境、组件库版本、用例总数
     - 测试结果统计：通过数、失败数、阻塞数、通过率、总耗时
     - 失败用例列表：按差异严重程度排序，支持按组件、差异类型筛选
   - 报告支持永久保存，生成唯一访问链接，支持团队内分享
2. **差异详情页**
   - 核心为**三图对比布局**，从左到右依次展示：基准图、当前测试截图、差异高亮图
   - 差异高亮图中，差异区域用红色标记，忽略区域用灰色遮罩标记，支持放大查看细节
   - 提供**滑动对比模式**：拖动滑块，实时叠加对比基准图与当前截图的重叠效果，直观查看差异
   - 展示详细差异信息：差异像素数量、差异占比、差异位置坐标、差异类型、环境参数
   - 提供一键操作按钮：
     - 「更新基准图」：确认差异为预期变更，一键用当前截图覆盖原有基准图
     - 「跳转到编辑器」：直接打开对应用例的DSL编辑器，调试修复问题
     - 「忽略本次差异」：标记本次差异为误报，不影响测试通过率
     - 「重新执行测试」：重新执行当前用例的测试
3. **报告导出能力**
   - 支持导出交互式HTML报告，可离线打开查看详情与差异对比
   - 支持导出JSON格式结构化测试数据，用于二次分析与系统集成
   - 支持导出PDF格式静态报告，用于归档与团队评审

#### 5.5 界面集成与测试中心
1. **全页面能力嵌入**
   - 组件列表：组件项展示测试状态标记，直观查看每个组件的测试通过率
   - 场景演示页：每个场景卡片展示测试状态，提供单场景测试、生成基准图按钮
   - DSL编辑器：顶部工具栏提供测试相关操作，预览区支持差异高亮预览
2. **独立测试中心页面**
   - 为测试工程师提供全量测试管理能力，分为4个核心tab页：
     1. **用例管理**：表格形式展示全量测试用例，支持筛选、批量操作、编辑、导入导出
     2. **测试执行**：支持选择用例集、配置测试环境，一键执行测试，实时展示执行进度
     3. **报告历史**：展示所有历史测试报告，支持按时间、版本、执行状态筛选，点击查看详情
     4. **配置管理**：全局测试配置，包括差异阈值、并发数、基准图存储路径、通知规则等

#### 5.6 CI/CD与自动化流水线集成
1. **CLI工具能力**
   - 提供独立的CLI命令行工具，支持在终端/流水线中执行核心操作：
     - `bi-playground test`：执行指定用例集/全量用例，生成测试报告
     - `bi-playground baseline`：批量生成/更新基准图
     - `bi-playground case export/import`：用例的导入导出
   - 支持配置执行参数，如并发数、差异阈值、输出路径、报告格式
2. **PR流水线集成**
   - 支持在GitHub Actions、GitLab CI、Jenkins等流水线中集成
   - 组件库提交PR时，自动触发全量视觉回归测试，测试不通过则阻断PR合并
   - 测试完成后，自动在PR评论中嵌入测试报告链接、通过率、失败用例的差异缩略图
3. **定时任务与告警**
   - 支持配置每日定时全量测试，监控组件库在不同浏览器、环境下的样式稳定性
   - 测试失败时，支持通过企业微信、钉钉、邮件发送告警通知，附报告链接与差异详情

### 模块6：辅助能力模块

## 五、技术实现方案
### 5.1 整体技术栈选型
| 技术领域 | 选型方案 | 选型说明 |
|----------|----------|----------|
| 核心框架 | React 18 + TypeScript 5.x | 与BI组件库技术栈保持一致，保证类型安全与渲染一致性 |
| 构建工具 | Vite 5.x | 提升开发与构建效率，支持模块热更新 |
| 代码编辑器 | Monaco Editor (@monaco-editor/react) | 与VS Code内核一致，提供完善的语法高亮、自动补全能力 |
| 状态管理 | Zustand | 轻量级状态管理，适配全局状态与组件状态管理 |
| 样式方案 | Ant Design 5.x + CSS变量 | 对标Ant Design交互体验，通过CSS变量实现主题切换 |
| 自动化截图与浏览器控制 | Playwright | 支持多浏览器、多视口、无头模式，API完善，适配CI/CD流水线 |
| 像素对比引擎 | pixelmatch + Resemble.js | 轻量高性能，兼顾对比精度与差异可视化能力 |
| 本地存储 | IndexedDB + localStorage | 适配基准图、DSL历史、用户配置的本地存储需求 |
| 云端存储 | 企业OSS/对象存储 | 用于团队基准图、用户配置、测试报告的云端存储 |
| 单元测试 | Vitest | 与Vite生态适配，保障核心功能稳定性 |

### 5.2 核心模块技术实现要点
1. **DSL实时渲染引擎**
   - 基于React的受控组件模式，将DSL配置作为state，修改后触发组件重渲染
   - 采用300ms防抖优化，避免频繁输入导致的重复渲染与性能损耗
   - 封装统一的渲染容器，隔离组件渲染环境，确保与业务项目效果一致
   - 实现错误边界捕获，组件渲染失败时不影响整个页面运行，同时展示友好错误提示

2. **视觉对比引擎**
   - 浏览器端：基于Canvas实现截图与轻量对比，用于编辑器实时差异预览
   - 服务端/CLI端：基于Playwright无头模式截图，pixelmatch做像素对比，生成差异图与结构化数据
   - 实现多环境一致性管控，截图时强制关闭动画、设置固定DPI、等待组件完全渲染，避免环境差异导致的误报
   - 实现忽略区域的像素掩码机制，对比时自动过滤忽略区域的像素差异

3. **Monaco Editor定制化**
   - 基于BI组件库的TypeScript类型定义，自动生成DSL的自动补全规则与语法校验规则
   - 实现JSON Schema校验，实时校验DSL的格式、参数类型、必填项
   - 封装自定义hover提示，对接API文档，实现参数说明的实时展示
   - 优化大文本编辑性能，启用虚拟滚动，避免大体积DSL编辑卡顿

4. **CI/CD集成方案**
   - 封装Docker镜像，将Playground CLI工具与Playwright环境打包，适配流水线运行环境
   - 实现制品库对接，基准图与测试报告作为制品存储，支持版本追溯

## 八、风险与应对措施
| 风险类型 | 风险描述 | 影响等级 | 应对措施 |
|----------|----------|----------|----------|
| 技术风险 | 不同浏览器、不同设备的渲染差异，导致基准图对比出现大量误报 | 高 | 1. 统一测试环境，基准图与测试截图必须在相同环境下生成<br>2. 提供可配置的差异阈值与忽略区域，降低误报率<br>3. 优先保障Chrome浏览器的测试准确性，其他浏览器做适配优化 |
| 性能风险 | 随着用例数量增加，全量测试耗时过长，影响迭代效率 | 中 | 1. 实现增量测试，仅执行DSL发生变更的用例<br>2. 优化并发执行策略，支持分布式测试执行<br>3. 提供用例集功能，支持按模块、优先级执行测试，无需每次全量执行 |
| 兼容性风险 | 组件库版本迭代后，历史DSL配置不兼容，导致用例渲染失败 | 中 | 1. 组件库迭代遵循语义化版本规范，非大版本不做破坏性变更<br>2. 提供DSL版本兼容转换工具，自动适配历史版本配置<br>3. 用例与组件库版本绑定，切换版本时自动加载对应版本的用例 |
| 协作风险 | 团队成员基准图不一致，导致测试结果冲突 | 中 | 1. 搭建云端基准图中心，作为唯一的基准图源，团队成员统一拉取<br>2. 基准图更新设置权限管控，仅指定人员可更新全量基准图<br>3. 基准图更新留痕，记录操作人与原因，支持追溯 |
| 业务风险 | 用户使用门槛高，无法快速上手测试功能 | 低 | 1. 提供预置用例，开箱即用，无需用户手动创建<br>2. 提供详细的使用文档与视频教程<br>3. 界面操作简化，核心功能一键执行，降低使用成本 |
