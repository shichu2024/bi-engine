## Problem

FEAT-010 完成了基础表格组件，但缺少以下关键交互能力：

1. **列管理 UI 位置不当**：当前列管理按钮在工具栏中，用户要求改为齿轮图标固定在表格最后一列的位置
2. **排序图标不够直观**：排序图标需要从当前的双箭头改为更直观的图标，显示在表头文字右侧
3. **筛选交互不符合预期**：筛选应改为漏斗图标点击触发，而非表头下方的输入框
4. **缺少分页**：大数据量时无法分页浏览，需要参考 antd 的分页组件
5. **列宽不可调整**：无法拖拽调整列宽
6. **表头文字溢出处理不当**：长标题超出列宽时缺少省略和 tooltip
7. **功能缺少开关控制**：列管理、排序、筛选缺少对外配置开关

## Goals

- **列管理增强**：齿轮图标固定在表格最后列位置，通过 `showColumnManager` 开关控制显隐
- **排序图标优化**：排序图标显示在表头文字右侧，通过 column `sortable` 开关控制
- **筛选交互重构**：漏斗图标点击触发筛选面板，通过 column `filterable` 开关控制
- **分页支持**：参考 antd 分页组件，控制每页展示数量，与列过滤和列排序联动
- **列宽拖拽**：支持拖拽调整列宽
- **表头文字省略**：超出列宽时省略展示并提供 tooltip，排序/筛选图标始终可见
- **排序/筛选/分页联动**：三者协同影响数据展示

## Scope

### In Scope

1. 列管理齿轮图标组件，固定在表格最后一列（虚拟列）
2. `showColumnManager` 表格级开关
3. 排序图标 UI 优化（表头文字右侧）
4. 筛选漏斗图标 + 下拉面板
5. 分页组件（参考 antd Pagination）
6. 分页与排序/筛选/列管理的联动
7. 列宽拖拽调整
8. 表头文字省略 + tooltip
9. 排序/筛选图标在表头中始终可见（不被截断）
10. 单元测试覆盖

### Out of Scope

- 虚拟滚动 / 大数据量性能优化
- 列固定 / 列冻结（sticky column）
- 树形数据展示
- 行选择 / 多选
- 行展开 / 折叠
- 可编辑单元格
- 导出 Excel / CSV

## Risks

| 风险 | 类型 | 缓解措施 |
|------|------|----------|
| 列宽拖拽与现有 table 布局的兼容性 | 技术 | 使用 `tableLayout: fixed` + `resize` 事件 |
| 分页状态与排序/筛选的组合复杂度 | 技术 | 数据管线：先筛选 → 排序 → 分页，每层独立 Hook |
| 筛选下拉面板定位可能被表格容器裁剪 | 技术 | 使用 Portal 渲染到 body |
| tooltip 在主题切换时的样式一致性 | 技术 | tooltip 使用 ThemeTokens |

## User Value

- BI 引擎使用者可以通过齿轮图标快速管理列显隐
- 表头排序和筛选图标更直观，符合用户使用习惯
- 分页支持让大数据量表格浏览体验更好
- 列宽拖拽和表头省略让表格展示更灵活

## Acceptance Criteria (Feature Level)

- **AC-F1**: 齿轮图标固定在表格最后一列，点击弹出列管理弹窗，移除的列不显示
- **AC-F2**: `showColumnManager` 为 false 时，不显示齿轮图标列
- **AC-F3**: 排序图标在表头文字右侧，column `sortable` 控制开关
- **AC-F4**: 筛选漏斗图标在表头，点击弹出筛选面板，column `filterable` 控制开关
- **AC-F5**: 分页组件参考 antd，支持切换每页条数和页码
- **AC-F6**: 筛选 → 排序 → 分页 数据管线正确
- **AC-F7**: 列宽可拖拽调整
- **AC-F8**: 表头文字超长时省略并显示 tooltip，排序/筛选图标始终可见
- **AC-F9**: 现有测试全部通过，新增测试覆盖 >= 80%
