# FEAT-002 Stories (V1.0 MVP)

## ST-001: 三栏式全局布局与导航系统

### 用户价值
作为使用者，我希望进入 Playground 后立即看到清晰的三栏式布局，左侧为组件导航，中间为内容区域，顶部为全局操作入口，让我能快速定位和使用 BI 组件。

### 验收标准
- [ ] 顶部导航栏展示品牌标识、版本切换器、主题切换、核心入口按钮
- [ ] 左侧组件列表区默认 240px 宽，支持折叠至 64px，支持拖拽调整宽度（200-400px）
- [ ] 主内容区自适应剩余宽度，最小宽度 800px
- [ ] 桌面端（>=1200px）完整三栏，平板端（768-1200px）左侧可收起，移动端（<768px）汉堡菜单
- [ ] 全局快捷键 Ctrl+K（搜索）、Ctrl+B（折叠侧栏）可用
- [ ] 主题选择、最近访问组件等偏好本地缓存，刷新后恢复

---

## ST-002: 左侧组件导航与检索系统

### 用户价值
作为使用者，我希望通过分类浏览或关键词搜索快速找到目标组件，并查看每个组件的测试状态。

### 验收标准
- [ ] 组件按 6 个一级分类展示（基础图表、高级分析图表、表格组件、布局容器、筛选交互、文本展示）
- [ ] 分类支持展开/折叠，折叠状态本地缓存
- [ ] 搜索框支持组件名称实时过滤，响应时间 <=200ms
- [ ] 组件列表项包含图标、名称、描述、测试状态标记
- [ ] 点击组件项，主内容区切换至该组件场景演示页，选中项高亮

---

## ST-003: 组件场景演示中心

### 用户价值
作为使用者，我希望选中组件后看到该组件的概览信息和多个典型业务场景，点击场景卡片能展开可交互的组件预览和 DSL 配置。

### 验收标准
- [ ] 组件概览区展示名称、版本标记、功能说明、核心能力标签
- [ ] 每个组件预置 >=3 个典型业务场景（基础用法、进阶用法、边界场景）
- [ ] 场景卡片展示标题、描述、预览缩略图
- [ ] 点击场景卡片展开详情：交互预览区 + DSL 代码片段（语法高亮）+ 操作工具栏
- [ ] DSL 代码片段支持一键复制，复制后显示成功提示
- [ ] 操作工具栏提供「在编辑器中打开」按钮，点击切换至 DSL 编辑模式

---

## ST-004: DSL 编辑器与实时渲染预览

### 用户价值
作为使用者，我希望在编辑器中修改 DSL 配置后能实时看到组件渲染效果，并且编辑器提供语法校验和自动补全。

### 验收标准
- [ ] 主内容区切换为左右等分可拖拽分栏布局（30%-70%）
- [ ] 左侧 Monaco Editor 支持 JSON 语法高亮、代码折叠、行号、格式化
- [ ] 实时语法校验：JSON 格式错误、参数类型错误标红，hover 展示错误提示
- [ ] DSL 修改后 300ms 防抖触发右侧渲染更新
- [ ] 渲染失败展示友好错误提示（含错误原因与位置）
- [ ] 编辑器工具栏：重置、格式化、复制 DSL、导出配置、保存为测试用例
- [ ] 预览区工具栏：全屏预览、主题切换、视口切换、禁用动画开关
- [ ] DSL 修改历史本地缓存最近 10 次

---

## ST-005: 基础视觉回归测试能力

### 用户价值
作为使用者，我希望能将组件场景保存为测试用例，生成基准截图，并在组件迭代后自动对比检测样式变更。

### 验收标准
- [ ] 支持创建测试用例（名称、关联组件、DSL 配置、Mock 数据为必填项）
- [ ] 支持在场景演示页和编辑器中保存当前配置为测试用例
- [ ] 基准图生成：锁定环境参数、禁用动画、等待渲染完成后截图
- [ ] 基准图存储在浏览器 IndexedDB，支持导出/导入
- [ ] 像素级对比：基于 pixelmatch 实现，支持配置差异阈值（默认 0.1%）
- [ ] 差异展示：三图对比（基准图、当前截图、差异高亮图）
- [ ] 测试结果：标记通过/失败，展示差异像素数、差异占比
- [ ] 一键更新基准图：确认差异为预期变更后覆盖
